◆ knownFuncNames_

std::unordered_map<std::string, std::string> psyche::ConstraintGenerator::knownFuncNames_
private

Undeclared functions require special treatment because they can still be defined later in the code. In such case, the function declaration we infer must match the function definition present. This is not a problem for the parameters but because return values may be discarded (i.e. expression statement) we keep track of the ones we know.
